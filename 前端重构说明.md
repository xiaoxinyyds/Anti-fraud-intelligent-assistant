# 前端登录界面分离 - 重构说明

## 概述
已将原有的单体Streamlit应用 (`front_end.py`) 重构为多页面应用，实现了登录界面与主功能界面的分离。

## 新的文件结构
```
Anti-fraud-intelligent-assistant/
├── frontend/                    # 新的前端目录
│   ├── app.py                  # 主应用入口（欢迎页面）
│   ├── pages/                  # Streamlit多页面目录
│   │   ├── 01_Login.py         # 登录/注册页面（独立界面）
│   │   └── 02_Main.py          # 多模态功能主界面
│   ├── components/             # 公共组件
│   │   ├── theme.py           # 主题管理（亮色/暗色模式）
│   │   └── __init__.py
│   ├── utils/                  # 工具函数
│   │   ├── api.py             # API客户端（认证、分析）
│   │   ├── session.py         # Session状态管理
│   │   └── __init__.py
│   └── run.py                 # 前端启动脚本
├── .streamlit/config.toml      # Streamlit配置（隐藏页面导航）
└── run_frontend.py            # 根目录启动脚本（方便使用）
```

## 主要变更

### 1. 登录界面 (`pages/01_Login.py`)
- 独立的认证界面，只包含注册/登录功能
- 注册时包含：角色选择、性别、风险灵敏度、监护人信息
- 认证成功后自动跳转到主界面
- 隐藏侧边栏，居中布局

### 2. 主功能界面 (`pages/02_Main.py`)
- 完整的多模态分析功能（文本、语音、图像）
- 显示侧边栏，包含用户信息和个性化设置
- 认证检查：未登录用户自动重定向到登录页面
- 保持原有的主题切换功能

### 3. 页面跳转机制
- 使用 `st.switch_page()` 实现页面跳转
- 登录成功后：`01_Login.py` → `02_Main.py`
- 退出登录后：`02_Main.py` → `01_Login.py`
- 未认证访问主界面时自动重定向

### 4. 状态管理
- 使用 `st.session_state` 共享认证状态
- 跨页面状态同步：`access_token`, `user_info`, 用户偏好设置
- 主题状态 (`theme`) 在所有页面中保持一致

### 5. 向后兼容性
- 原 `front_end.py` 文件保持不变（作为备份）
- 现有API接口保持不变
- 模拟分析功能在后端不可用时仍可工作

## 启动方式

### 方法1：使用根目录启动脚本（推荐）
```bash
python run_frontend.py
```

### 方法2：直接进入frontend目录
```bash
cd frontend
python run.py
```

### 方法3：直接使用streamlit命令
```bash
cd frontend
streamlit run app.py
```

## 测试说明

### 认证流程测试
1. 访问应用，默认显示登录页面
2. 注册新用户或使用现有账户登录
3. 验证成功跳转到主界面
4. 验证侧边栏显示用户信息
5. 测试退出登录功能

### 功能完整性测试
1. 文本分析：输入可疑文本，点击分析
2. 语音分析：上传音频文件，测试分析
3. 图像分析：上传图片，测试OCR和场景分析
4. 高级选项：测试深度分析功能
5. 主题切换：测试亮色/暗色模式

### 页面跳转测试
1. 未登录直接访问主界面 → 应重定向到登录页面
2. 登录成功后 → 应跳转到主界面
3. 退出登录后 → 应跳转到登录页面
4. 浏览器刷新 → 应保持认证状态

## 注意事项

1. **后端依赖**：认证和分析需要后端API服务（`http://localhost:8000`）
2. **模拟模式**：后端不可用时，使用模拟分析功能
3. **文件上传**：音频文件支持 .wav/.mp3/.m4a，图片支持 .jpg/.jpeg/.png
4. **主题切换**：主题状态在页面跳转时保持
5. **浏览器兼容性**：建议使用Chrome/Firefox最新版本

## 故障排除

### 1. 导入错误
- 确保在项目根目录或frontend目录中运行
- 检查Python路径包含当前目录

### 2. 页面跳转失败
- 确保使用Streamlit 1.20.0+ 版本（支持 `st.switch_page()`）
- 检查页面文件名和路径是否正确

### 3. 认证状态丢失
- Streamlit的session_state在页面刷新时可能重置
- 重要操作建议在本地存储备份

### 4. 后端连接失败
- 检查后端服务是否运行（`http://localhost:8000`）
- 系统会自动切换到模拟分析模式
